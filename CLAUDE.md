## Project snapshot

- **Product:** Ranná Správa — Slovak daily newsletter (lang=`sk`), morning format.
- **Site:** https://rannasprava.sk — static, GitHub Pages, custom domain (CNAME), `.nojekyll`, no build step.
- **Owner:** ADAM (git user).
- **Backend:** `api.rannasprava.sk` — Node.js + Caddy on VPS. Only endpoint: `POST /subscribe` → Brevo API. No SSR.
- **Subscriber counter** in `index.html` is hardcoded base **1100** + localStorage delta. Not real count. Two occurrences: hero section and footer.
- **OG image worker:** `og.rannasprava.sk` — Cloudflare Worker (satori + resvg-wasm). Query params: `?n=[issue]&t=[URL-encoded title]&d=[YYYY-MM-DD]`. Returns 1200×630 PNG. Used on all archiv pages.

## Repo layout

- `index.html` — SPA landing. Renders issue list client-side from `issues.js` → `ISSUES` array. Newest issue goes top of array.
- `issues.js` — source of truth for all issue metadata. Every new issue added here first.
- `landing.html` — separate landing variant.
- `vydania/[N]/index.html` — static per-issue HTML, fully crawlable. Issues 48–86 published (see anomalies below).
- `vydania/[N]/sources.md`, `[N]-podcast.txt`, `social-posts.md`, `edits.md` — per-issue sidecars.
- `emails/[N]-brevo.html` — Brevo newsletter export per issue.
- `archiv/DD/MM/YYYY/` — full indexable editorial pages, one per issue. Canonical public archive URLs. Self-canonical (not redirecting to vydania/N/). Generated by `generate-archive-date-pages.ps1`.
- `temy/[slug]/index.html` — topic landing pages (6 topics). Generated by `generate-topic-pages.ps1`. Up to 10 issue cards + evergreen intro from `temy-intros/[slug].html`.
- `temy-intros/[slug].html` — editable UTF-8 intro fragments for each topic page. Raw diacritics safe (read with `-Encoding UTF8`).
- `issues/`, `share/`, `podcastrecs/`, `socials/` — content + distribution assets.
- `robots.txt` — allows GPTBot, ClaudeBot, PerplexityBot. Has `Sitemap:` directive.
- `sitemap.xml` — 81 URLs: homepage + 39 vydania + 42 archiv pages (as of issue #86).
- `llms.txt`, `llms-full.txt` — GEO infra. Filters out issues 1–3 (no vydania pages).
- `feed.xml` — RSS feed, 30 most recent issues.

## Automation scripts (PowerShell + Python)

- `update-market-snapshot.ps1` — markets block.

- `update-podcast-embed.ps1` — podcast embed.

### Site regeneration (run via `publish.ps1`, not individually)
- `publish.ps1` — **master script**. Run with `-Issue N`. Runs all 6 generators in sequence. Auto-detects latest issue if `-Issue` omitted.
- `generate-archive-date-pages.ps1` — builds full `/archiv/DD/MM/YYYY/` editorial pages. v3 (2026-04-24): self-canonical, NewsArticle+BreadcrumbList schema, OG image, cold-open extraction, prev/next nav, Slovak day/month names. Excludes issues 1–3 (no vydania). 42 pages.
- `generate-static-archive.ps1` — static archive links injected into `index.html`.
- `generate-sitemap.ps1` — scans `vydania/` + `archiv/` folders, writes `sitemap.xml`. 81 URLs. Excludes folder `492`. No UTF-8 BOM (plain UTF-8). All vydania have `<lastmod>` extracted from HTML (JSON-LD or title tag). All archiv have `<lastmod>` from path.
- `generate-feed.ps1` — RSS feed, 30 most recent issues.
- `generate-llms.ps1` — `llms.txt` + `llms-full.txt`. Filters issues without vydania pages.
- `generate-topic-pages.ps1` — 6 topic pages in `temy/`. Loads intro from `temy-intros/[slug].html`. 10-card cap with "more" link to archiv. Excludes issues 1–3. Slovak topic slugs: `slovensko`, `biznis`, `tech`, `svet`, `sport`, `zdravie`.

- `ping-indexnow.ps1` — Bing/Yandex ping. Flags: `-Issue N` (single), `-Url "..."` (arbitrary), `-All` (all sitemap URLs).

## Publishing workflow

**Phase 1 — Issue content (AI does this, one commit):**
1. Add issue object to top of `ISSUES` array in `issues.js`
2. Create `vydania/[N]/index.html` and `vydania/[N]/sources.md`
3. Run: market snapshot → weather snapshot → overlap check → Brevo export → podcast embed
4. Update `how-we-do-ranna-sprava.md` with publish note
5. Commit core files + push `main`

Core files: `vydania/[N]/index.html`, `vydania/[N]/sources.md`, `issues.js`, `index.html` (cache-buster only), `how-we-do-ranna-sprava.md`.

**Phase 2 — Asset regeneration (rannaspravaposts skill handles automatically):**
Skill runs `publish.ps1 -Issue N` → all derived files regenerated → git add/commit/push → `ping-indexnow.ps1 -Issue N`.

AI does NOT run individual generator scripts — the skill handles everything.

Derived files: `archiv/DD/MM/YYYY/`, static archive in `index.html`, `sitemap.xml`, `feed.xml`, `llms.txt`, `llms-full.txt`, all `temy/*/index.html`.

**Quality checks before Phase 1 push:**
1. Issue number, date, title, preview correct in `issues.js`.
2. Appears in Home, Archive, Reader views.
3. No duplicates or overwrites (overlap checker).
4. `how-we-do-ranna-sprava.md` updated.
5. Push and verify public page source.

## Standing rules (non-negotiable)

- **Every commit updates docs.** `how-we-do-ranna-sprava.md` mandates a doc update on every push. AI must update relevant `.md` before commit/push.
- **SEO/GEO fixes marked in all `.md` files.** Every SEO/GEO fix annotated in `GEO-AUDIT-REPORT.md` and `how-we-do-ranna-sprava.md`. Cross-session tracking so fixes aren't re-audited.
- **Typography frozen** unless user asks. No drift in font family, size, weight, spacing. `design-and-structure-spec.md` is SOT.
- **No topic/category labels** (`Slovensko`, `Biznis`, `Tech`, `Šport`) in archive metadata or as visible kickers. Archive cards show only: issue number, date, title, preview.
- **Archive URLs** use `/archiv/DD/MM/YYYY/`. These are canonical — home + archive listings link here, not to `/vydania/[N]/`.
- **`robots.txt`** lives at repo root — must never 404.
- **PowerShell 5.1 encoding rule:** Scripts without BOM are read as Windows-1252. Never put raw Slovak diacritics in PS here-strings. Use HTML entities (`&aacute;`, `&iacute;`, `&mdash;`, etc.) for HTML output, or `\uXXXX` for JSON strings, or `[char]0xXXXX` for PS variables. Read diacritic files with `-Encoding UTF8`.
- **Issues 1–3** have no `vydania/` pages. All generators must filter them via `Test-Path "vydania\$n\index.html"`.
